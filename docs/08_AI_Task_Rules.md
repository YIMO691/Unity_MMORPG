# AI 任务执行规则

## 目标

AI 只能作为小范围工程任务的执行和审查助手，不能无限制重写整个 MMORPG Demo。

## 基本规则

1. 每次任务必须有明确目标。
2. 每次任务必须列出允许修改的文件或目录。
3. 每次任务必须列出禁止修改的范围。
4. 每次任务必须包含可执行验收标准。
5. 生成代码时不能硬编码 API Key、账号、密码或真实服务密钥。
6. 修改架构、技术栈、协议格式前必须先写设计说明并获得确认。
7. 涉及服务端权威逻辑时，客户端只能上报意图，不能决定最终结果。
8. 涉及 Lua 热更新时，高频逻辑仍应留在 C# 或服务端。
9. 每次有意义的工程变更都应同步相关文档。

## 推荐任务粒度

- 一个 Unity UI 面板。
- 一个 protobuf 协议组。
- 一个服务端接口。
- 一个 Lua 模块。
- 一个配置表结构。
- 一个 bug 的最小修复。
- 一组相关单元测试。

## 不推荐任务

- 一次生成完整 MMORPG。
- 一次重构全部客户端。
- 一次重写服务端架构。
- 让 AI 自动选择并引入大量框架。
- 让 AI 在没有验收标准的情况下改代码。
